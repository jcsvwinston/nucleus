Lo que enseña

  • Páginas MVC públicas para catálogo y captación de leads.
  • Vistas MVC con sesión de usuario y dashboard protegido.
  • API JSON con envelopes homogéneos {data,count}.
  • Panel /admin con CRUD, system pulse y feature flags.
  • OpenAPI servido en runtime y outbox visible para operación.

Endpoints que merece la pena probar

GET /articles renderiza el catálogo MVC con solo artículos publicados.
GET|POST /contact captura leads desde una vista HTML clásica.
GET /api/articles lista artículos.
POST /api/articles crea contenido y además encola un evento en outbox.
GET /api/articles/live-flag cambia según la feature flag del admin.
GET /api/leads y POST /api/leads muestran otro recurso de negocio.
POST /api/demo/outbox y POST /api/demo/outbox/drain enseñan el carril outbox.
POST /api/demo/tasks activa el carril de jobs si configuras Redis.

Recorrido sugerido

  • Recorre /articles como zona pública y usa /contact para dejar un lead.
  • Entra en /app/login con las credenciales de demo y abre el dashboard.
  • Ve a /admin, crea o edita artículos y leads.
  • Abre /admin/system para ver outbox, cluster y jobs.
  • Lanza POST /api/demo/outbox y luego POST /api/demo/outbox/drain.
  • Si arrancas Redis, verás también queues y schedules en el panel.

Consejo: este ejemplo está pensado para enseñar el objetivo del framework: juntar la ergonomía de Django con la experiencia de producción estilo Encore.
