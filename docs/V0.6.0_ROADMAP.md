# GoFrame v0.6.0 Roadmap

Reference date: 2026-04-05.
Status: Current (target release planning).
Target release window: May 2026.

## Release Theme

`v0.6.0` focuses on enterprise hardening and extensibility, not only adding more commands.

Primary outcome:

- secure-by-default runtime posture
- deeper production observability
- stable capability-based plugin contracts for extension
- stronger CI confidence across SQL engines

## Scope Strategy

## In Scope (P0 - must ship)

1. Plugin SDK v1 (stable contract)
- formal capability contract for extension points (`mail.send`, `queue.publish`, `subscription.create`, `webhook.deliver`)
- unified request/response envelope and exit semantics for external plugins
- compatibility bridge from legacy mail plugin naming (`goframe-mail-<driver>`) to capability runtime
- plugin developer documentation + official examples across at least two capabilities
- compatibility guarantees documented for `v0.6.x`

2. Observability v2 (DB + jobs)
- DB telemetry semantics (query duration, error rates, pool stats)
- background job telemetry (enqueue, start, success, retry, failure)
- correlation path from HTTP request context to async job execution where applicable
- deploy checks updated to validate critical observability configuration

3. Security v2 (policy and guardrails)
- rate limiting dimensions extended (burst, route-level, role-level)
- stronger `check --deploy` validations for security-sensitive config
- review and tighten defaults around session/cookie/CORS/CSRF behavior
- threat-focused tests for sensitive input/output paths

4. CI SQL matrix for critical workflows
- CI coverage for SQLite, PostgreSQL, and MySQL
- required suites for migration, fixture, health, and shell-related behavior
- clear separation between fast smoke and full matrix jobs

## Stretch (P1 - if capacity allows)

1. Shell UX improvements
- persistent history
- multiline input editing

2. External-adopter docs polish
- operational runbooks for deployment and incident response basics

## Out of Scope for v0.6.0

- broad new feature families unrelated to hardening/extensibility
- disruptive API redesigns without migration guidance

`v0.6.0` can be released only when all are true:

- [ ] all P0 checklist items completed or explicitly descoped with approval
- [ ] `go test ./...` green on `main`
- [ ] release rehearsal script green (`bash scripts/release/rehearse_rc.sh`)
- [ ] required CI matrix jobs green (SQLite/PostgreSQL/MySQL)
- [ ] changelog and docs aligned with shipped behavior
- [ ] no open high-severity release blockers

## Risk Register

1. Scope creep beyond hardening objectives
- mitigation: strict P0 freeze after 2026-04-10

2. CI matrix instability increases iteration time
- mitigation: split fast smoke vs full matrix, stabilize per suite

3. Plugin contract churn breaks early adopters
- mitigation: publish SDK contract first, version it, then implement against it

4. Security hardening introduces regressions
- mitigation: negative tests + staged rollout + deploy-check guidance
